# Contributing to shelldialogs

Thank you for your interest in contributing to **shelldialogs**.

We welcome bug reports, feature requests, documentation improvements, and code contributions.

---

## Reporting Bugs

Before creating a bug report:

* Ensure you are using the latest version.
* Search existing Issues to avoid duplicates.

When submitting a bug report, please include:

* Operating system and version
* Java version
* Library version
* Steps to reproduce
* Expected behavior
* Actual behavior
* Stack trace (if applicable)

Use the provided Issue template for bug reports.

---

## Suggesting Enhancements

We appreciate ideas that improve usability, API design, performance, or documentation.

When proposing a new feature:

* Clearly describe the problem it solves
* Provide a proposed API or usage example (if possible)
* Explain why it fits the project scope

Use the provided Feature Request template.

---

## Pull Requests

### Workflow

1. Fork the repository
2. Create a new branch from `main`
3. Implement your changes
4. Add or update tests if applicable
5. Ensure the project builds successfully
6. Open a Pull Request

### Guidelines

* Keep changes focused and minimal
* Follow existing code style and naming conventions
* Avoid breaking public API without discussion
* Write clear commit messages

Example commit message:

```
Add support for multi-selection menu
Fix NPE in SelectionMenu builder
```

---

## Code Style

* Java 21 compatible
* Follow existing formatting
* Prefer clear, readable APIs over clever solutions

---

## Questions

If you are unsure about a design decision, open a Discussion before implementing major changes.

Thank you for helping improve shelldialogs.
